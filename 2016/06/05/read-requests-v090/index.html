发表于 2016-06-05   |   分类于 编程   |  

从早期代码看过来的一个体验就是其实大神也有年轻的时候，也犯过一些错误，埋了一些不高明的BUG，比如我注意到Request对象的sent属性其实并没有起到作用，它本应该在调用send方法之后被置为True，然而我找了整个代码都没找到有设为True的地方，也就是说，无论如何都会发起请求。所幸我看到这个版本的Issue里面有人提到了这个问题。

目标

v0.9.0的主要改动就是 SSL 认证了。

修正了请求的URL路径问题

新加了两个属性full_url和path_url取代了之前的_build_url方法。full_url返回完整路径，path_url返回域名之后的子路径。构建请求时将path_url传给urlopen，这样可以改善性能，因为完整的url信息已经包含在连接对象里了。看具体实现：

有一点值得注意，我发现作者在构造字符串时喜欢将各部分放在列表里，再用空字符串连接起来，这要换我来写就直接target_str += component了，不知道这样有什么好处。
